# Полный список всех свойств технологического журнала 1С

**Источник:** ИТС 8.3.27, раздел 3.24.2.4.2 "Перечень свойств"

## Общие свойства (присутствуют во всех или большинстве событий)

- `name` - Имя события
- `level` - Уровень важности события (TRACE, DEBUG, INFO, WARNING, ERROR)
- `duration` - Длительность события в сотнях микросекунд
- `durationus` - Длительность события в микросекундах
- `depth` - Уровень события в стеке текущего потока
- `process` - Наименование приложения (имя файла загрузочного модуля)
- `OSThread` - Номер потока операционной системы
- `ClientID` - Идентификатор соединения с клиентом по TCP
- `SessionID` - Номер сеанса, назначенный текущему потоку
- `Usr` - Имя пользователя информационной базы
- `AppID` - Идентификатор клиентской программы
- `ConnID` - Идентификатор соединения с информационной базой
- `Context` - Контекст исполнения (стек вызовов)
- `TransactionID` / `Trans` - Идентификатор активности транзакции

## Свойства SQL-событий (DBMSSQL, DBPOSTGRS, DBORACLE, DB2, DBV8DBENG, DBDA, EDS)

- `sql` - Текст оператора SQL
- `planSQLText` - План запроса
- `Rows` - Количество полученных записей базы данных
- `RowsAffected` - Количество измененных записей базы данных
- `Database` - Путь к используемой базе данных
- `Dbms` - Имя СУБД (DBMSSQL, DBPOSTGRS, DBORACLE, DB2, DBV8DBENG, DBDA, DBMySQL и др.)
- `Dbpid` - Идентификатор соединения сервера с СУБД (для DBMSSQL, DBPOSTGRS, DB2, DBORACLE)
- `DBCopy` - Имя используемой копии базы данных
- `NParams` - Количество параметров оператора SQL (для DBV8DBENG)
- `MDX` - Текст MDX-запроса к OLAP системе (для EDS)
- `DBConnID` - Идентификатор соединения с СУБД внешнего источника данных (для EDS)
- `DBConnStr` - Строка соединения с внешним источником данных (для EDS)
- `DBUsr` - Имя пользователя СУБД внешнего источника данных (для EDS)

## Свойства запросов SDBL

- `Query` - Текст запроса на встроенном языке
- `Sdbl` - Текст запроса на встроенном языке модели базы данных
- `Database` - Путь к используемой базе данных
- `Dbms` - Имя СУБД
- `DBCopy` - Имя используемой копии базы данных

## Свойства исключений (EXCP, EXCPCNTX)

- `Exception` - Наименование программного исключения
- `Descr` - Пояснения к программному исключению
- `Context` - Контекст возникновения исключения
- `Func` - Наименование выполняемого действия (имя функции/метода)
- `Line` - Номер строки в файле
- `File` - Имя файла, в котором было сгенерировано событие
- `Module` - Имя модуля
- `OSException` - Описание исключения операционной системы

## Свойства ошибок запросов (QERR)

- `Query` - Текст запроса на встроенном языке
- `QueryFileds` - Перечень полей запроса, в которых обнаружены значения NULL
- `Descr` - Описание ошибки

## Свойства блокировок (TLOCK, TTIMEOUT, TDEADLOCK)

- `Locks` - Список управляемых транзакционных блокировок
- `Regions` - Имена пространств управляемых транзакционных блокировок
- `WaitConnections` - Список соединений, с которыми идет столкновение по блокировкам
- `lka` - Поток является источником блокировки
- `lkp` - Поток является жертвой блокировки
- `lkpid` - Номер запроса жертвы
- `lkaid` - Номера запросов источника
- `lksrc` - Номер соединения источника
- `lkpto` - Время ожидания жертвы
- `lkato` - Время блокировки источником
- `DeadlockConnectionIntersections` - Список пар транзакций, образующих взаимную блокировку (для TDEADLOCK)

## Свойства соединений (CONN)

- `ConnID` - Идентификатор подключения
- `Server` - Имя сервера
- `Port` - Номер основного сетевого порта процесса
- `SyncPort` - Номер вспомогательного сетевого порта процесса
- `Connection` - Номер соединения с информационной базой
- `hResultOLEDB`, `hResultNC2005`, `hResultNC2008`, `hResult2012` - Коды возврата при попытке соединения через провайдеры

## Свойства сеансов (SESN)

- `Session` / `SessionID` - Номер сеанса
- `Nmb` - Номер сеанса (альтернативное имя)
- `seanceID` - Идентификатор сеанса «1С:Предприятие»

## Свойства процессов (PROC)

- `ProcID` - Идентификатор процесса
- `PID` - Идентификатор процесса операционной системы
- `ProcessName` - Наименование процесса
- `p:processName` - Имя серверного контекста
- `srcProcessName` - Имя общих данных на момент создания
- `Finish` - Причина завершения процесса
- `RunAs` - Режим запуска процесса (приложение или сервис)

## Свойства вызовов (CALL, SCALL)

- `Method` - Имя вызываемого метода (или номер метода интерфейса)
- `MName` - Имя удаленно вызываемого метода
- `IName` - Имя передаваемого интерфейса
- `Interface` - Идентификатор интерфейса
- `CallID` - Идентификатор вызова
- `RetExcp` - Исключение, возникшее при выполнении вызова сервера
- `DstClientID` - Идентификатор клиента назначения
- `Memory` - Объем памяти в байтах, занятой, но не освобожденной за серверный вызов
- `MemoryPeak` - Пиковое значение за вызов памяти в байтах

## Свойства кластера (CLSTR)

- `Event` - Наименование действия, выполняемого кластером
- `Cluster` - Номер основного порта кластера серверов
- `Server` - Имя сервера
- `IB` - Имя информационной базы
- `Ref` - Имя информационной базы
- `Connections` - Количество соединений, которым не хватило рабочих процессов
- `ConnLimit` - Установленное максимальное количество соединений на один рабочий процесс
- `Infobases` - Количество информационных баз, которым не хватило рабочих процессов
- `IBLimit` - Установленное максимальное количество информационных баз на один рабочий процесс
- `DstAddr`, `DstId`, `DstPid`, `DstSrv` - Назначенный рабочий процесс
- `SrcAddr`, `SrcId`, `SrcPid`, `SrcSrv`, `SrcURL` - Предпочтительный рабочий процесс
- `MyVer`, `SrcVer` - Версии состояния кластера
- `Registered`, `Obsolete`, `Released` - Информация о рабочих процессах
- `Reason` - Причина недоступности рабочего процесса
- `Request` - Идентификатор запроса на соединение
- `ServiceName` - Имя сервиса кластера серверов
- `ApplicationExt` - Уточнение требования назначения функциональности
- `NeedResync` - Требуется синхронизация данных сервера
- `NewServiceDataDirectory`, `OldServiceDataDirectory` - Пути каталогов данных сервиса

## Свойства серверных контекстов (SCOM)

- `Server` - Имя сервера
- `ServerComputerName` - Имя рабочего сервера
- `procURL` - Адрес процесса сервера
- `agentURL` - Адрес текущего процесса агента сервера

## Свойства административных действий (ADMIN)

- `Admin` - Имя администратора кластера или центрального сервера
- `Action` - Текстовое описание выполняемой операции

## Свойства памяти (MEM, LEAKS, ATTN)

- `Sz` - Объем занятой процессом динамической памяти (в байтах)
- `Szd` - Изменение объема динамической памяти
- `cn` - Количество занятых процессом фрагментов динамической памяти
- `cnd` - Изменение количества фрагментов динамической памяти
- `MemoryLimits` - Заданные пределы памяти (для ATTN)
- `ExcessDurationSec` - Продолжительность превышения памяти, в секундах (для ATTN)
- `ExcessStartTime` - Временная отметка превышения памяти (для ATTN)
- `FreeMemory` - Объем свободной оперативной памяти (для ATTN)
- `TotalMemory` - Объем оперативной памяти компьютера (для ATTN)
- `SafeLimit` - Безопасный расход памяти за один вызов (для ATTN)
- `Info` - Количество попыток установки соединения с процессом, завершившимся с ошибкой (для ATTN)
- `PID` - Идентификатор процесса главного менеджера кластера (для ATTN)
- `ProcessId` - Уникальный идентификатор главного менеджера кластера (для ATTN)
- `ServerId` - Уникальный идентификатор сервера (для ATTN)
- `Url` - Адрес главного менеджера кластера (для ATTN)

## Свойства лицензирования (LIC, HASP)

- `res` - Действие, выполненное системой лицензирования (для LIC)
- `HaspID` - Идентификатор ключа защиты (для HASP)

## Свойства полнотекстового поиска (FTEXTUPD, FTS, FTEXTCHECK, INPUTBYSTRING)

- `State` - Начало или завершение операции обновления индекса (для FTEXTUPD)
- `AvMem` - Доступная память в момент вывода (для FTEXTUPD)
- `BackgroundJobCreated` - Формирование индекса в фоновом процессе (для FTEXTUPD)
- `MemoryUsed` - Максимальный размер динамической памяти (для FTEXTUPD)
- `FailedJobsCount` - Количество фоновых процессов, завершившихся с ошибками (для FTEXTUPD)
- `TotalJobsCount` - Количество созданных фоновых процессов (для FTEXTUPD)
- `JobCanceledByLoadLimit` - Признак отмены процесса из-за предела загрузки (для FTEXTUPD)
- `MinDataId` - Минимальный идентификатор индексируемых данных (для FTEXTUPD)
- `Files`, `FilesCount`, `FilesTotalSize`, `Folder` - Информация о файлах (для FTEXTUPD, если logfiles=true)
- `Time` - Время вывода записи (для FTEXTUPD) или время выполнения операции (для FTS)
- `File` - Имя файла с проблемой (для FTEXTCHECK)
- `Info` - Информация о сбое (для FTEXTCHECK)
- `Result` - Результат проверки (для FTEXTCHECK)
- `Separation` - Включено или нет разделение (для FTEXTCHECK)
- `SepId` - Индекс области разделения (для FTEXTCHECK)
- `Word` - Слово, если определено (для FTEXTCHECK)
- `FindByString` - Имя объекта конфигурации (для INPUTBYSTRING)
- `Text` - Текст, введенный при вводе по строке (для INPUTBYSTRING)
- `findTicks` - Время поиска в базе данных, в мсек (для INPUTBYSTRING)
- `ftextTicks` - Время полнотекстового поиска, в мсек (для INPUTBYSTRING)
- `ftextSearchCount` - Количество вызовов полнотекстового поиска (для INPUTBYSTRING)
- `ftextResultCount` - Количество ссылок, найденных полнотекстовым поиском (для INPUTBYSTRING)
- `SearchByMask` - Используется поиск без полнотекстового поиска (для INPUTBYSTRING)
- `tooManyResults` - В индексе слишком много ссылок (для INPUTBYSTRING)
- `FillRefsPresent` - Используется кеш ссылок (для INPUTBYSTRING)
- `jobId` - Идентификатор фонового задания индексации (для FTS)
- `logFrom`, `logTo` - Временные метки порции данных (для FTS)
- `fixedState` - Новая отметка в файле состояния (для FTS)
- `recordCount` - Количество записей в порции данных (для FTS)
- `totalRecords` - Общее количество записей (для FTS)
- `tableCount` - Количество таблиц (для FTS)
- `tableName` - Имя таблицы (для FTS)
- `tableCode` - Код таблицы (для FTS)
- `tableRef` - Ссылка на объект конфигурации (для FTS)
- `metaDataId` - Уникальный идентификатор объекта метаданных таблицы (для FTS)
- `recordRef` - Внутренняя ссылка на запись таблицы (для FTS)
- `fullKey` - Полный ключ для поиска записи (для FTS)
- `reindexCount` - Количество таблиц, отправленных на переиндексацию (для FTS)
- `skippedRecords` - Сколько записей пропущено из-за ошибок (для FTS)
- `parallelism` - Сколько фоновых заданий допустимо запускать параллельно (для FTS)

## Свойства хранилища двоичных данных (STORE)

- `Id` - Идентификатор объекта
- `Size` - Размер объекта
- `StorageGUID` - Уникальный идентификатор хранилища
- `BackupFileName` - Имя файла с резервной копией
- `BackupBaseFileName` - Имя файла с полной резервной копией
- `BackupType` - Режим резервного копирования (0 - полное, 1 - дифференциальное)
- `MinimalWriteSize` - Пороговый размер данных
- `ReadOnlyMode` - Признак режима «только чтение»
- `UseMode` - Признак использования хранилища

## Свойства сборщика мусора (SDGC)

- `InstanceID` - Уникальный идентификатор хранилища
- `Method` - Текущее действие сборщика мусора (Compact, Analyze)
- `FilesSize` - Общий размер хранилища
- `UsedSize` - Размер используемого места
- `CopyBytes` - Общий размер скопированных значений
- `LockDuration` - Длительность блокировки хранилища

## Свойства внешних компонент (ADDIN)

- `Classes` - Имена объектов внешней компоненты
- `Location` - Расположение внешней компоненты или идентификатор объекта
- `MethodName` - Имя метода внешней компоненты
- `Message` - Сообщение, указанное во время вызова события
- `Source` - Источник, указанный при вызове события
- `Type` - Имя типа
- `Result` - Результат выполнения операции (0 - успешно, 1 - с ошибкой)
- `Crashed` - Равно 1, если операция была завершена с ошибкой
- `ErrorDescr` - Текст исключения при возникновении ошибки

## Свойства системных событий (SYSTEM)

- `Class` - Имя класса, в котором сгенерировано событие
- `Component` - Имя компоненты платформы
- `File` - Имя файла, в котором было сгенерировано событие
- `Line` - Номер строки в файле
- `Txt` - Текст информационного сообщения

## Свойства журнала регистрации (EVENTLOG)

- `FileName` - Имя файла с формируемым индексом
- `CpuTime` - Процессорное время, затраченное на индексирование
- `OSThread` - Номер потока, выполняющего индексацию
- `PacketCount` - Фактическое количество пакетов, записанных в файл индекса

## Свойства видеосвязи (VIDEOCALL, VIDEOCONN, VIDEOSTATS)

- `Connection` - Выбранная пара ICE-кандидатов
- `Status` - Состояние соединения
- `StreamType` - Тип потока (видео, аудио или экран)
- `Value` - Статистика по потоку данных
- `cpu` - Уровень загрузки процессора
- `QueueLenght` - Количество запросов в очереди процессора
- `InMessage` - Полученные сообщения от тонкого клиента
- `OutMessage` - Отправленные сообщения в тонкий клиент
- `Direction` - Направление потока (входящий/исходящий)
- `Type` - Тип потока (видео/аудио)

## Свойства распознавания речи (STT, STTAdm)

- `Id` - Идентификатор сессии потокового распознавания
- `Key` - Ключ локальной сессии
- `modelID` - Идентификатор модели распознавания речи
- `Path` - Путь к компоненту модели
- `AudioEncoding` - Формат аудио фреймов
- `Frames` - Количество аудио фреймов
- `Contexts` - Количество элементов дополнительной грамматики
- `ContextsOnly` - Признак использования только дополнительной грамматики
- `Recording` - Признак записи аудио потока
- `Status` - Статус результата обработки компонента
- `Phrase` - Количество элементов дополнительной грамматики
- `rx:Acoustic`, `rx:Grammar`, `rx:Language`, `rx:Location`, `rx:SampleRate`, `rx:Version` - Выбранные системой параметры
- `tx:Acoustic`, `tx:Grammar`, `tx:Language`, `tx:Location`, `tx:SampleRate`, `tx:Version` - Запрошенные параметры

## Свойства веб-сервисов (VRSREQUEST, VRSRESPONSE)

- `URI` - Ресурс, к которому производится обращение
- `Method` - HTTP-метод обращения к ресурсу
- `Headers` - HTTP-заголовки запроса/ответа
- `Body` - Размер в байтах тела запроса/ответа
- `Status` - Код состояния HTTP (для VRSRESPONSE)
- `Phrase` - Текстовая фраза, соответствующая коду состояния (для VRSRESPONSE)

## Свойства интеграции (SINTEG, EDS)

- `SrvcName` - Имя объекта метаданных сервиса интеграции (для SINTEG)
- `ExtSrvcUrl` - Адрес внешнего сервиса интеграции (для SINTEG)
- `ExtSrvcUsr` - Имя пользователя внешнего сервиса интеграции (для SINTEG)

## Свойства почты (MAILPARSEERR)

- `MessageUid` - Уникальный идентификатор почтового сообщения
- `Method` - Метод объекта ИнтернетПочта (GET, GETHEADERS, SETRAW)

## Свойства сертификатов (WINCERT)

- `certificate` - Описание проверяемого сертификата
- `errorCode` - Код ошибки Windows API

## Свойства истории данных (DHIST)

- `description` - Текст, поясняющий выполняемое действие

## Свойства загрузки конфигурации (CONFLOADFROMFILES)

- `Action` - Текстовое описание выполняемой операции

## Свойства фоновых заданий

- `Report` - Имя объекта метаданных выполняемого отчета

## Свойства ввода по строке (INPUTBYSTRING)

- `FindByString` - Имя объекта конфигурации
- `Text` - Текст, введенный при вводе по строке
- `findTicks` - Время поиска в базе данных, в мсек
- `ftextTicks` - Время полнотекстового поиска, в мсек
- `ftextSearchCount` - Количество вызовов полнотекстового поиска
- `ftextResultCount` - Количество ссылок, найденных полнотекстовым поиском
- `SearchByMask` - Используется поиск без полнотекстового поиска
- `tooManyResults` - В индексе слишком много ссылок
- `FillRefsPresent` - Используется кеш ссылок

## Свойства с префиксами (t:, p:, rx:, tx:)

- `t:applicationName` - Идентификатор клиентской программы
- `t:clientID` - Идентификатор соединения с клиентом по TCP
- `t:computerName` - Имя клиентского компьютера
- `t:connectID` - Идентификатор соединения с информационной базой
- `p:processName` - Имя серверного контекста
- `rx:*` - Выбранные системой параметры распознавания речи
- `tx:*` - Запрошенные параметры распознавания речи

## Дополнительные свойства

- `Host` - Имя компьютера
- `Val` - Значение, смысл зависит от значения параметра Func
- `Err` - Тип консольного сообщения (0 - информационное, 1 - об ошибке)
- `Calls` - Количество обращений клиентского приложения к серверному приложению через TCP
- `InBytes` - Количество данных, прочитанных с диска за время вызова (в байтах)
- `OutBytes` - Количество данных, записанных на диск за время вызова (в байтах)
